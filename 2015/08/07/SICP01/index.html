环境

  • 用来保存值与符号的关联，而后又能够提取出这些值，更精确的说应该是全局环境；运算符同样包含在全局环境里，并且将相应的指令序列作为与之关联的“值”；环境说扮演的角色就是表达式中各个符号的意义；环境还有点类似于资料库（上下文），对于程序里面的各种符号做出解释

递归

  • 在自己的工作步骤中，包含着调用这个规则本身的需要；使用递归的思想可以简单的描述深度嵌套的情况；到现在我都有点害怕的=递归这种东西，总觉得很难去理解，自己调用自己，我潜意识里面认为自己调用自己不会有改变，其实不然，递归调用在每一次调用之后都会改变输入的参数，直到不再调用自身可以确切的得到答案位置
  • 递归过程与递归计算过程
    • 递归过程是指在计算过程中有出现自己调用自己；而递归计算的过程指的是上一步的计算执行依赖于下一步计算的结果，就是有使用都推迟计算
  • 递归计算过程与迭代计算过程的区别
    • 迭代计算过程每个人都需要保存纸条(每个人手上纸条的数量还不一样)，迭代计算结束只是在传递纸条
    • 迭代计算过程能用明确的变量，保存计算机过程中的状态；而递归计算过程除了保存形式变量中的信息，计算机还要保存一些不可见的信息，也就是都有哪些过程是被推迟计算的信息
  • 怎么写递归计算过程的程序？怎么写迭代计算过程的程序？如何进行选择？
    • 递归计算过程实际上有点类似于对计算的拆解
      • 1.首先要明确递归的边界条件，也就是递归到了什么地方会有确切的解，而不是调用自己
      • 2.要明确调用自己的意义，调用自己一定是为了能够达到边界条件而努力，利用调用自己得出来的结果组合出需要的答案，要明白自己最终一定会得到结果；就好像下面的例子中调用自己相加是根据斐波那契数列的定义进行组合的
      • 3.调用自身之后会返回什么？首先参数会减小，其次返回的是整个过程本身，可以画成树状图来进行分析，一目了然

  • 递归目前有两种形式

    • 1.第一种是斐波那契数列那样递归是表达自己的定义
    • 2.第二种是像求阶乘那样对问题进行分解

    • (Define (FIB N)
    (if (< N 2)
        N
        (+ (FIB (- N 1))
            (FIB (- N 2)))))

  • 递归的实质就是模拟一个过程的完成，然后通过缩小参数来控制自动演算完剩下的所有过程

语法的糖衣

  • 比如要定义一个平方运算的实现，规规矩矩的写需要很多行代码，但是通过语法糖可以采用简单的写法却实现相同的效果，说白了就是一种简化的写法

    //定义一个平方运算
//方案一：原始的写法，符合Define的标准定义模式，其中Lambad用于构建一个过程，square与过程绑定在一起
(Define square (Lambad (x) (*x x)))

//方案二：语法糖的写法，本质是上面的写法，这是一个命名绑定的过程；上面一个方案其实也可以认为是语法糖的展开
(Define (square x) (* x x))

函数与过程

  • 函数是描述一件事情的特征，过程是描述如何去做这件事；函数说平方根的定义是这样的，而过程是平方根是要怎么样才可以求出来；编写的程序指导着过程的进行，通过语言来形式化的描述这一个过程

  • 构建复杂系统是要解决的核心问题；构建计算机程序是一个抽象的过程，不像其他的领域，存在种种约束

黑盒抽象

  • 可以利用黑箱子去祝贺出更大的黑箱子；箱子的输入与输出不一定就是数字，还可以是一个过程，输入是一个过程，输出也可以是一个过程；就好比求函数不动点的方法，用来求平方根只是它的一种特例用法，它还可以用在很多其他的方面上
  • 黑盒抽象的精髓在于你不需要知道利用的东西到底是内建的支持还是别人自定义的过程

组合式

  • 组合式由操作符和运算对象组成
  • 运算对象本身也可以是一个组合式
    0 lisp当中采用前缀表达式，操作符放在前面，运算对象放在后面；被括号起来就代表这是一个组合式，意味着要将运算符作用于运算对象
  • 前缀表达式的使用规则
    • 括号的最左边一定是一个操作符，可以是+-，也可以是define；右边是运算对象，对于+-而言，运算对象只能有两个，当然运算对象本身可以是一个组合式，因此可以嵌套；define用法相同，可以用来绑定符号与数字，也可以用来绑定符号与运算过程，使用方法同+—；但是define在绑定过程上有一个语法糖的存在，把自己会传入的参数提取出来，放在一个括号里面，后面再跟着过程主体

  • 使用S表达式最大的优点就是可以不需要去关心所谓的运算顺序，因为括号的存在使得根本就不需要去在意这种规则，无论从哪一个开始计算其结果都是一样的

    //绑定数字
(Define x 2)
//绑定过程
(Define (square x) (* x x))
//Define后面一旦遇到一个括号，就会进入语法糖模式
(Define A (* 5 5)) //绑定一个数字
(Define (D) (* 5 5)) //绑定一个过程
//上面两个结果是一样的，但是解释器的执行是不一样的
//在解释器输入A会返回25；输入(A)，会提示错误，因为A不是一个运算符，一个操作过程
//在解释器输入(D)会执行D过程，然后返回25；直接输入D会提示错误

  • define可以绑定不止一个过程，在第一个括号之后，后面的都是要尽心绑定的一系列过程，绑定完执行表达式的时候就是从绑定的第一个过程开始执行

    (Define (sqrt x)
  (Define (improve guess)
    (average guess (/ x guess)))
  (Define (good-enough? guess)
    (< (abs (- (square guess) x))
        .001))
  (Define (try guess)
    (if (good-enough? guess)
        guess
        (try (improve guess))))
  (try 1))

代换模型

  • 对于基础过程的定义
    • 没有必要太过于纠结一些基础过程的实现，就好像+-如何实现一样，去深入研究这些队研究无益

公共模式

  • 给过程命名，然后可以将过程作为参数尽心传递，过程参数会在相应位置展开，就像是将一大段代码打包作为参数进行传递一样，这一块还需要好好的研究
  • 当很多的代码共享一种抽象的基础模型，就可以抽象出一种公共模式；define中的参数，可以当做数字来使用，那就是数字参数，也可以当做过程来使用，那就是过程参数，这是非常神奇的一个地方
  • 使用公共模式可以抽象，使得大多数的运算变得易于维护，只需要修改公共模式的实现，就可以做到修改所有使用公共模式来实现的运算
  • 把过程作为参数进行传递可以在被调用的位置直接展开，非常简单

Lambad

  • 生产匿名过程
    • 可以使用Lambad关键字生成一个匿名的过程，由于是匿名的过程，所以写完之后一定就要直接调用了，因为是匿名的，所以无法获得引用，如果不在写完之后直接调用，就会失去意义
