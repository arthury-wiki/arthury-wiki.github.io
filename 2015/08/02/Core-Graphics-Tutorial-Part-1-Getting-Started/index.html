Core Graphics Tutorial Part 1: Getting Started(翻译)

前言

  • 想象一下你应经完成了你的app并且它运行的很好，但是界面缺少风格。你可以在你的Photoshop里面绘制不同尺寸的图片并且希望Apple不要出@4的retina屏幕
  • 或者，你可以深谋远见的使用Core Graphics来使用代码创建一张图片，这样无论是在什么尺寸的屏幕上都可以完美的展示
  • Core Graphics是Apple的大型矢量绘图框架，强大的api，需要学习的东西有很多。但是不要去害怕，接下来的三个部分会带你深入浅出的学习，并且在最后你可以在自己的app上面创建进人的图形
  • 这是一个全新的系列，使用现代的方式来教学Core Graphics。这个系列已经更新到了Xcode6和Swift，并且覆盖了一些新的功能，比如@IBDesignable以及@IBInspectable这些可以让CoreGraphics更加有趣和简单

绘制按钮

  • 你将要对按钮进行如下图所示的绘制
  • 为了在Core Graphics中画一个形状，你可以定义一个形状来告诉 Core Graphics要跟踪的线条，就像两条直线一样，或者需要被填充的线条。如果熟悉在Photoshop重点插图或者矢量形状，那么你将会很容易的理解轨迹
  • 关于轨迹有三点是需要知道的
    • 一个轨迹可以被画出来以及填充
    • A stroke outlines the path in the current stroke color
    • 当前的填充颜色会填充封闭的路径
  • 一种简单的方式去创建Core Graphics轨迹是通过一个便利的类，称之为UIBezierPath，也就是所谓的贝塞尔曲线。这个类可以让你轻松地使用便利友好的api创建出轨迹，无论你想要创建以线条、曲线、长方形、或者是一些列连接的点为基础的路径，都可以做到

在Core Graphics的背后

  • 每一个UIView有一个绘图上下文，在视图被转换到设备的之前，所有的绘图工作都会完成渲染
  • iOS通过调用drawRect(_:)来更新绘图上下文，无论何时视图需要被更新都要调用这个方法，这个方法会在一些情况被调用：
    • 当视图第一次载入的时候
    • 顶部的其他视图被移走的时候
    • 当视图的hidden属性被改变的时候
    • 你可以通过直接调用setNeedsDisplay()和setNeedsDisplayInRect()方法来调用这个方法
  • 注意：所有在drawRect(_:)中进行的绘图工作都载入到视图的绘图上下文当中。如果你开始在drawRect(_:)之外进行绘图，就像你会在最后一部分教程中做的一样，你将会不得不创建你自己的绘图上下文
  • 你还没有使用Core Graphic，因为UIKit已经封装了许多的Core Graphic函数。UIBesizerPath就是对CGMutablePath的封装，这是对底层的Core Graphics API进行封装
  • 注意：绝不要直接调用drawRect(_:)方法，如果你的视图没有被更新，那么就使用setNeedsDisplay()；setNeedsDisplay()自身并没有调用drawRect(_:)，但是它会将视图标记为’dirty’，会在下一次屏幕的更新循环当中使用drawRect(_:)进行重绘，就算你在同一个方法当中调用setNeedsDisplay()五次，实际上也只会调用drawRect(_:)一次

将代码载入上下文

  • Core Graphics使用画家模式
  • 当你载入上下文，它就像是在画画。你设置轨迹并且填充它，然后设置另外的轨迹在上一个轨迹的上面并且填充它。你不能够改变那些还没有被设置的像素，但是你可以在它们上面进行画画
  • 下面这张图片来自苹果的文档，描述了它是如何工作的。就像你正在一个画布画图，你绘图的顺序是非常关键的
  • 从图上可以看出，先绘制的会放在下面，后绘制的会放在上面
  • 你的加号按钮将要放置在蓝色圆圈的上面，所以首先要绘制出蓝色的圆然后才是加号
  • 你可以为加号按钮绘制两个矩形，但是绘制一个轨迹然后使用期望的厚度去将其画出来

  • 将以下代码添加到drawRect(_:)的末尾，绘制出加号按钮需要的水平短线：

    //set up the width and height variables
//for the horizontal stroke
let plusHeight: CGFloat = 3.0
let plusWidth: CGFloat = min(bounds.width, bounds.height) * 0.6

//create the path
var plusPath = UIBezierPath()

//set the path's line width to the height of the stroke
plusPath.lineWidth = plusHeight

//move the initial point of the path
//to the start of the horizontal stroke
plusPath.moveToPoint(CGPoint(
x:bounds.width/2 - plusWidth/2,
y:bounds.height/2
))

//add a point to the path at the end of the stroke
plusPath.addLineToPoint(CGPoint(
x:bounds.width/2 + plusWidth/2,
y:bounds.height/2))

//set the stroke color
UIColor.whiteColor().setStroke()

//draw the stroke
plusPath.stroke()

点和像素

  • 追溯到最开始的iPhone，点和像素占据了相同的空间并且是相同的大小，使得它们完全就是相同的东西。当retina屏幕的iPhone出现之后，突然像素的数量变成了点的四倍
  • 相似的，iPhone6 Plus又一次增加像素的数量
  • 注意：接下来的是概念，实际设备上的像素可能稍有不同。注意学习使用paint code来辅助自己快速绘图，参考资料
    -这里有12x12的网格，点有灰色和白色。第一个iPad2中点和像素是一样对应的；第二个是2x的retina显示屏，这里4个像素组成一个点；第三个是一个3x的retina屏幕，这里9个像素组成一个点
  • 你绘制的线有3个点的高度，线条是从轨迹的中心开始画的，所以轨迹的两边分别都是1.5个点
  • 下面这张图片展示3个点的高度在各个设备中被绘制出来的情况。你可以看到iPad2和iPhone6 Plus都有划线穿过半个像素，这是无法被完成的。所以，iOS就会在这一半被填充的像素使用一种模糊不清的处理方式(anti-aliases)
  • 实际上，iPhone6 Plus有着非常多的像素，所以模糊影像看起来就不是那么的明显了，即便如此你应该确定你的app在不同设备上的表现情况。但是如果你是为非retina显示屏的设备比如iPad2或者iPad mini做开发，你应该极力避免出现anti-aliases的情况，因为像素点不多，所以模糊的效果非常的明显
  • 如果你有奇怪厚度的直线，你需要在给它们定位的时候+0.5或者是-0.5来防止出现anti-aliasing的情况。如果上面的图表，你将会看到在iPad2上面出现的一个点被分成两半的情况，在这种情况下就应该要把点往上移动半个像素，在iPhone 6上面，不需要做移动，在iPhone Plus上面，同样移动半个像素

  • 在drawRect(_:)方法中，修改moveToPoint与addLineToPoint为：

    //move the initial point of the path
//to the start of the horizontal stroke
plusPath.moveToPoint(CGPoint(
x:bounds.width/2 - plusWidth/2,
y:bounds.height/2 + 0.5
))

//add a point to the path at the end of the stroke
plusPath.addLineToPoint(CGPoint(
x:bounds.width/2 + plusWidth/2,
y:bounds.height/2 + 0.5
))

  • iOS现在会非常清晰的渲染出这些直线在三个设备的屏幕上

  • 为了画出完美填充像素的直线，你可以通过绘制和填充一条贝塞尔曲线来代替一条直线，并且可以使用视图的contentScaleFactor来计算矩形的宽度或者高度(也就是厚度)。而不是直接根据终点与起点直接画出轨迹，填充只会在轨迹内部进行，不会对外部进行调整

@IBInspectable–自定义StroyBoard属性

  • 除了一个加号按钮，你还需要一个减号按钮

  • 完成的代码如下所示

    import UIKit

@IBDesignable
class PushButtonView: UIButton {

  @IBInspectable var fillColor: UIColor = UIColor.greenColor()
  @IBInspectable var isAddButton: Bool = true

  override func drawRect(rect: CGRect) {

    var path = UIBezierPath(ovalInRect: rect)
    fillColor.setFill()
    path.fill()

    //set up the width and height variables
    //for the horizontal stroke
    let plusHeight: CGFloat = 3.0
    let plusWidth: CGFloat = min(bounds.width, bounds.height) * 0.6     

    //create the path
    var plusPath = UIBezierPath()

    //set the path's line width to the height of the stroke
    plusPath.lineWidth = plusHeight

    //move the initial point of the path
    //to the start of the horizontal stroke
    plusPath.moveToPoint(CGPoint(
      x:bounds.width/2 - plusWidth/2 + 0.5,
      y:bounds.height/2 + 0.5))

    //add a point to the path at the end of the stroke
    plusPath.addLineToPoint(CGPoint(
      x:bounds.width/2 + plusWidth/2 + 0.5,
      y:bounds.height/2 + 0.5))

    //Vertical Line
    if isAddButton {
      //move to the start of the vertical stroke
      plusPath.moveToPoint(CGPoint(
        x:bounds.width/2 + 0.5,
        y:bounds.height/2 - plusWidth/2 + 0.5))

      //add the end point to the vertical stroke
      plusPath.addLineToPoint(CGPoint(
        x:bounds.width/2 + 0.5,
        y:bounds.height/2 + plusWidth/2 + 0.5))
    }

    //set the stroke color
    UIColor.whiteColor().setStroke()

    //draw the stroke
    plusPath.stroke()

  }

}

即兴的数学课

  • 我们短暂的插入这个教程，回顾一下高中时候上的数学课。就像道格拉斯说的–不要惊慌！:]
  • 载入上下文的绘图代码是在单位圆的基础上进行的，单位圆就是半径为1的圆。
  • 上图中红色的部分你的弧将会从哪里开始与结束，顺时针方向绘图。你将会从3π/4的地方开始画弧，画到π/4结束。
  • 弧度常常在代码中代替度数，使用弧度来思考是很有必要的，这样你就不要每一次都转换为度数了。之后你需要指出弧线的长度，这个时候度数就会起到作用。
  • 一个弧在单位圆中的长度就像是角度在弧度中的测量一样。举个栗子，看看上面的图表，弧线的长度从 0º到90º是π/2。为了计算出弧线的长度，需要将在单位圆中的弧长等比例放大到真实圆的半径当中。

  • 为了计算红色弧长的长度，我们需要进行一下的计算：

    2π – end of arrow (3π/4) + point of arrow (π/4) = 3π/2

  • 度数是

    360º – 135º + 45º = 270º

//1 - first calculate the difference between the two angles
//ensuring it is positive
let angleDifference: CGFloat = 2 * π - startAngle + endAngle

//then calculate the arc for each single glass
let arcLengthPerGlass = angleDifference / CGFloat(NoOfGlasses)

//then multiply out by the actual glasses drunk
let outlineEndAngle = arcLengthPerGlass * CGFloat(counter) + startAngle

//2 - draw the outer arc
var outlinePath = UIBezierPath(arcCenter: center, 
                                  radius: bounds.width/2 - 2.5,
                              startAngle: startAngle, 
                                endAngle: outlineEndAngle,
                               clockwise: true)

//3 - draw the inner arc
outlinePath.addArcWithCenter(center, 
                     radius: bounds.width/2 - arcWidth + 2.5,
                 startAngle: outlineEndAngle, 
                   endAngle: startAngle, 
                  clockwise: false)

//4 - close the path
outlinePath.closePath()

outlineColor.setStroke()
outlinePath.lineWidth = 5.0
outlinePath.stroke()
