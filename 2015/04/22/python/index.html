python整体的学习架构

语言特性

  • python属于动态语言
    • python不需要知道数据类型，所以不用写类型声明，非常的爽，直接想要使用什么就去用什么便可
    • 虽然说python使用变量不需要声明，但是一个变量(标识符)一定要事先经过赋值才可以进行使用，否则就会出现NameError

  • python属于强类型语言

    • 变量类型一旦确定，就不会出现进行自动转换，比如int自动转换成float并不会进行，也就是所谓的隐式转换，那么在弱类型语言里面int类型的变量什么时候会隐式转换成float

    • 隐式转换(在C语言中)

      • 不同类型的数据在运算时计算机会先自动将其转换成同一类型之后再进行计算，主要是char->int->double<-float，这几个类型在几乎在所有类型的语言里面都是存在的

      • 隐式转换并不会改变变量的数据类型，变量的数据类型一旦确定下来，如果是在C语言当中，是没有办法再进行修改的，但是在python当中仍然是可以进行修改，因为python是解释运行的，每遇到一条代码命令，就进行编译，对于同一个变量，进行多次不同类型数据的赋值，是会被解释成不同的数据类型

        a = 1
        a = “c”
        a = 3.7

      • 如上代码最后a的类型是float

      • 如此看来隐式转换只是针对于变量里面的数据，而不是这个变量，所以改变类型的只是数据，而不是变量本身；这样看来无论是显示(强制)转换与隐式(自动)转换都是针对于数据而言，而不是针对于变量而言

列表与数组

  • 列表看起来就像是打了鸡血的数组，因为比数组要强大
    • 列表可以通过方法进行动态伸缩
    • 列表本身同样可以作为列表的元素
    • 列表允许混合数据
    • 往一个列表里面插入数据有两种方法，一种是使用insert方法，另一种是重新创建列表；无论是哪一种方法都会有适应的场景
  • 列表分别有可变与不可变的两种情况，一种是使用[]来创建，这一种是可变的；另一种是用()来创建，这一种是不可变的，也叫做元组(tuple)，怎么与swift这么像，注意元组是不可变的列表

python当中的转义字符

  • 无论是单引号还是双引号，都可以用来创建一个字符串，只要注意不可以混合使用便可
  • 同样与其他的语言一样，python也可以使用\来进行转义

python是大小写敏感的

冒号是本体

  • for循环的遍历，这个与其他语言的遍历没有太大的区别
for a in b:
        printf(a) 
        //注意movie后面别忘记了加上一个冒号，这个是python很重要的一个特征
  • if后面有跟上条件并不需要加上括号，结束之后运行语句也不需要加花括号，只需要加上冒号便可；for循环也是一样的
movie = ['a','b','c',['s','d','e','w','q']]
for i in movie:
    if isinstance(i,list):
                for inset in i:
                    print(inset)
    else:
        print(i)

对于isinstance的使用

  • 可以是用来判断对象的类型，比如isinstance(movie,list)用来判断movie是否是一个列表类型

在python中创建函数

  • def 函数名(参数):函数代数组，果然python中冒号本体
  • 写一个递归函数来实现列表输出
    def print_list(the_list):
        for insetl in the_list:
            if isinstance(insetl,list):
                print_list(insetl)
            else:
                print(insetl)
    //这里直接使用了递归来实现全部输出，关于递归的使用这一块，我有必要好好的进行学习一下    


+ 递归函数的递归深度是有限制的，当然如果想要嵌套的更深完全可以自己修改上限

python的注释方式

+ “”“这里是python的注释”“”，其实这也可以自己当成是一个字符串，就是一个没有用来赋值的一个字符串，所以就算用单引号或者是双引号来注释都是可以的，只要自己注意格式便可
+ 除了使用引号，还可以使用#号来进行注释，一个#就可以直接注释到语句尾
+ 注释貌似不能够有中文字符，这是编译器的问题还是python就是这样规定的呢

如何导入python的一个模块

import print_list #不需要.py后缀
  • 导入之后之后函数不能直接调用，就像print_list；这样调用会出现错误，就是python解释器找不到这个函数，因为命名空间的问题；正确的访问方式是模块名.print_list(movie)，这里的模块名并不是文件夹的名字，而是程序的文件名，比如程序是在print_list.py里面，那么print_list就是模块名
  • 当然也可以更为特定的导入， from 模块名 import print_list，这样就可以直接print_list来使用了，但是这样会覆盖掉自己在当前文件写的同名函数，所以不太好用

良好的代码设计模式

  • 对一个函数进行增强，并不需要再写一个全新的函数，这样会导致选择过多，比较合理的解决方案是通过提供参数来，通过参数来控制函数的行为，这样才是比较理想的
  • 一点一点为用户考虑的设计想法实在是太棒了，让我学到了设计一个函数api如何才能够周全的考虑的问题，通过多样的参数与缺省值来控制函数的行为，而不是一点小修改就去设计一个新的api，这样会导致有太多的api，非常的糟糕

关于命名空间

  • 每一个函数都会有一个命名空间

python中即强大又灵活的特性

  • python在运行之前并不会检查代码的合法性，也就是不管代码如何都一定会被执行，这才是重点
  • 不去编译，直接就开始运行，虽然非常的灵活强大，但是同时也会使得一些bug成为落网之鱼

对于缺省值的理解

  • 比如一个函数要求提供很多个接口，如果有的参数并不需要传入进来，每个都去写没有又是一件非常麻烦的事情，这样可以通过缺省值来解决这个烦恼，有了缺省值，有的参数没有写，那就会自动使用这个缺省值

关于print()函数自动换行功能的关闭

  • print("\t",end = '')，这一段函数的功能就是在输出一个制表符之后禁止自动换行，这其实是print的一个可选参数，用来关闭其自动换行的功能

直接在python的shell里面也可以进行目录的切换

  • 通过os模块可以直接获得当前所在的目录以及切换到任意的目录
  • os.getcwd()//获得当前的路径
  • os.chdir('../../current')//切换到指定的目录

python的异常捕获(异常处理机制)

  • 在运行的时候出现了异常可以直接捕获到，然后对这个异常进行处理，python的异常处理非常的强大，允许错误发生，然后给你一个机会来恢复，太酷了，一旦发现了异常，可以直接进行修复之后再运行
  • try与except语句，作为处理异常的机制非常的强大
try:
    #你想要执行的功能代码
except:
    #try里面的代码发生了错误之后就会执行这里面的代码
    pass #可以用这个pass命令直接无视掉try里面的代码
  • try语句非常的使用，不仅可以让代码更加容易的进行阅读，而且出问题时更容易修正
  • except还可以指定捕捉的错误类型，也就是只在特定的错误发生的时候才会去执行except下面的代码
try:
    data = open('sketch.text')
    for each_line in data:
        #if not each_line.find(':') == -1:
        try:
            (role, line_spoken) = each_line.split(':',1)
            print(role,end='')
            print(' said: ',end='')
            print(line_spoken,end='')
        except ValueError:
            pass #x 空语句，什么都不执行
    data.close()#关闭之前打开的文件
except IOError:
    print('The data file is missing!')
