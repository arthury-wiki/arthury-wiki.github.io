Application architecture

前言

  • 这里介绍的应用架构相关的知识，这一块的内容是应用层学习的很关键的一块内容，当然这里面设计到的思想可以应用在非常多的地方，关键就是学习的思想，以及自己平时做APP考虑不周的地方

iOS应用架构学习

iOS应用架构谈

  • 架构师的角色非常的重要，让自己拥有一个架构师的实力，这样今后自己去做开发的时候就不会一团乱麻了，在前期就把祸根全部清扫掉，为后期的高速发展奠定基础
  • 成为一个优秀的架构师是非常必要的
  • 架构的实现可否以用于对自己所学知识的管理上面，使用一个好的架构来管理与维护自己的知识库，让自己不再苦逼
  • 这里几个名词讲解一下：
    • 业务方：就是负责苦逼写代码的
  • 现在自己写出的app的缺陷
    • 代码混乱不堪：现在写代码至少知道方法名要取的合理，并且将多个功能点分开，一个功能对应一个方法；会造成混乱的一个原因是大量引用了别人的库，自己不是能够很好的驾驭，这也是一个很重要的原因
    • 依赖关系复杂
