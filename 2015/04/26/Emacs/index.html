常用快捷键

  • Meta键的映射，目前键盘上并没有找到
  • C-a 移动到一行的开头；C-e 移动到一行的结尾
  • M-a 移动到句首；移动到句末
  • ESC } 下一段落
  • ESC { 上一段落
  • C-x C-s(save-buffer)，保存缓冲区文件
    • 出现问题之后可以采用C-s(停止接受输入) C-q(激活控制流)
    • 也可以使用enable-flow-control命令来进行解决
  • ESC x 可以用来输入没有键盘快捷键的命令，输入之后按下return
  • C-x C-w(write-file)，重命名并保存文件，不命名则使用原来的名字
    • 这个命令同样可以用来创建一个不可修改文件的副本，重命名保存到不同的路径便可，这样就可以有一个可以修改的私有版本了
  • C-x C-c，退出Emacs
  • 在文件末尾插入文件的快捷键
    • 先按下esc，松开之后再按下>(就是shift+.)，然后再输入C+x，稍等一下出现提示之后，再按下i键便可，会自动跳到文件末尾插入
    • 另外esc + >也可以单独用来作为跳到文件末尾的快捷键
    • esc + < 可以调到文件的开头
  • C-x C-f 打开文件
  • C-x C-v 重新选择要打开的文件
  • C-u 8 C-v 向下滚动8行，当然数字是可以改变的
  • C-g 可以取消不想要执行的命令，注意是g
  • C-h可以调出帮助文档，当然需要相应的参数
    • t 教程
    • k 按键释义
    • f 函数释义
    • i info文档
  • C-x i 在当前为止插入文件
  • C-q C-l 在当前为止插入分页符
  • C-x ] 移动到下一个分页符所在位置
  • C-x [ 移动到下一个分页符所在位置
  • C-/ 撤销操作，undo，类似的操作还有C-y，还有ESC y可以恢复删除环里面所有的文本
  • C-r redo，也就是取消撤销的操作
  • C-l 重新绘制屏幕，当屏幕变得有点混乱之后就可以使用这个来整理一下屏幕
  • 删除操作
    • C-d 删除一个字符
    • ESC-d 删除的是整个单词(具体的效果要自己在实战当中进行总结)
    • C-k 可以做到删除一整行的作用，注意使用的时候一定要把光标移动到开头，
      删除是从光标之后开始的
    • ESC-k 删除光标之后的句子直到段尾
    • C-x DEL 删除光标之前的句子直到段首
    • C-w 删除文本块，就是从光标开始，要设置一个文本文本标记(Mark set)，
      mark set与当前的光标所包围的区域就是文本块，这时候就是可以使用C-w进
      行删除，使用C-space来设置文本标记点
      • 可以互换两个标记的地方，使用C-x C-x组合键来实现
      • C-w其实是剪切，剪切下来的文本在适当的地方使用C-y来进行黏贴
      • 大范围的标记
        • ESC h 自动标记当前光标所在的段落
        • C-x h 自动标记整个缓冲区的内容
        • 如果已经有使用页面的分页符，那么可以使用C-x C-p来快速标
          记整个页面
    • ESC-C-k 效果是删除从行首(不是整个文件的首部)开始到光标所在的位置
      (这一按下esc之后还有一条横线，不要忘记这个)
    • 被删除之后的数据会进入到删除环里面，等着随时被恢复，实际上删除环里
      面可以容纳30次左右的删除，当然也可以自己通过命令来扩充容量，但是一般不会有这样的必要
  • 复制与删除
    • 使用文本标记辅助自己选中想要复制的文本块，然后按下ESC-w进行复制，
      之后在恰当位置使用C-y便可
  • 没有键盘快捷键的命令(输入命令的时候同样可以使用Tab进行补全)
    • auto-fill-mode 开启/关闭自动换行模式，看一下状态栏上面有没有fill，
      但是只对当前的缓冲区有效果
    • goto-line n 跳转到第n行
    • goto-char n 跳转到文件的第n个字符
    • 重复执行某一个命令
      • ESC n command n代表你想要执行的命令的次数，command代表你想要执行的命令
      • demo: esc 500 C-n 就是向下换行500次
  • 简化编辑的小技巧
    • C-t 可以交换前后两个字符的位置，一般在打英文的时候会使用到，现在想想纯英文的世界实在是太美好了，不用去在意输入法的问题
    • C-x C-t可以交换两个文本行
    • 以下操作全部都统一把光标移动到首字母上去，因为之后对光标头开始到单词尾结束产生作用
      • ESC c 首字母大写
      • ESC l 将首字母改成全小写
      • ESC u 将首字母改成全大写
      • 以上是对当前的字母进行操作，还可以对前一个单词进行操作
      • 在以上的操作命令之前加上ESC-便可(注意这里的横线是要加上去的)
  • 撤销所有修改，返回最后一次保存的状态
    • revert-buffer 这将会让文件直接返回到最后一次存盘的状态，是恢复当前的文件
  • 关闭掉辅助窗口的快捷键
    • C-x 1(数字)

自动备份的文件

  • Emacs会在第一次保存某一个文件的时候创建一个备份，就是平时看到的文件名后面加一个~，但是并不能直接从备份文件中进行恢复的命令
  • 使用备份文件覆盖当前文件的操作
    • 直接对~文件进行编辑，然后使用C-x C-w 文件名 RETURN来进行覆盖
    • Emacs支持保存多份的备份，每一次存盘之前都会有一个备份，可以无限备份下去，但是最后是备份最近的几次就好，这就是版本控制模式(VC mode)

出现了非正常退出Emacs的时候怎么办？

  • Emacs有一个auto-save的功能，其实就是自动保存的功能，这个会生成一个#
    文件名#的文件，这个文件在Emacs非正常退出之后就会出现，就是自动保存的文
    件内容，如果出现了非正常的退出就可以从这个文件里面恢复
  • 使用recover-this-file命令可以从自动保存的文件里面进行恢复，非常强大
  • 可以使用aoto-save-interval命令来修改自动保存的间隔，是以敲击键盘达
    到多少次来作为判断的标准
  • 如果对Emacs文件进行了大量的文件删除工作就会就不会进行自动保存，此时
    要进行一次存盘才会重新开始自动保存；或者使用ESC 1 ESC x auto-save RETURN(这里是数字1)来强制自动保存

自动补全功能这一块

  • 使用打开文件的快捷键的时候，可以打出开头的几个字母，注意这几个字母一
    定要是开头的并且要能够唯一识别一个文件的，然后按下Tab键便可自动补全；
    其实如果打出的首字母不够确认，那么Emacs就会给出一个列表，对照着列表可
    以再打几个字母，然后再按下Tab， 这种使用Tab来进行补全的办法在shell环境
    下都是可以使用的，没有任何的问题，就比如cd路径的时候也可以这样做，非常
    的方便
  • 注意这里是大的字母可以无视大小写，系统会进行自动修正，所以没有必要特
    意去写大写

      • 这里介绍一下emacs的feature机制

        • 在emacs的lisp代码里面，你可以require一个可以提供某一个功能
          的library，使用以下的代码

          (require 'the-feature "the-library-file"
<soft-flag>)

        • 在没有.emacs文件的情况下，这些配置就写在init.el文件里面

        • 这个library通过如下代码表示自己提供的功能

          (provide 'the-feature)

          v

        • snippets/ #yasnippet的自定义模板保存的位置
        • elpa/ #通过ELPA下载的插件所保存的位置
        • lisp/ #就是加载各个插件的初始化文件的位置，每一个插件都有
          一个与自己对应init文件，用来定制一些自己的配置，也就是原来放
          在.emacs下的对于常见插件的自定义，都可以拆分到各自的init文件
          下面了
        • editing-utils/ #文本编辑用的一些小工具
        • custom-themes/ #自定义的主题
        • custom-dicts/ #自定义的auto-complete词典，不重要

        • init-utils.el

          最主要的作用是提供了一个宏 after-load ，供后续的各初始化函
          数使用。这个函数来自 Purcell ，目的是把一些相互依赖的
          feature的加载顺序理顺，例如feature A依赖于feature B，则可
          以写成 (after-load ‘B ‘A) ，这样如果错误地在B之前require了
          A也不会影响正常启动。

        • require-package函数的作用

          • 判断某一个包是否已经安装，如果没有就从elpa提供的插件源当
            中去安装，如果是没有收录的就无能为力了
          • 通过这个函数就可以很方便的去安装各种各样的插件了，不用自
            己再去特意的下载下来，然后在.emacs里面配置这么麻烦了
          • 只需要在每个插件对应的init文件里面做相应的配置便可

emacs的查找功能

  • 递增查找
    • 使用C-s命令来进行查找，当前高亮的如果不是自己想要的哪一个，就继续
      使用C-s来进行查找，此时不需要再次输入要查找的字符串，会自动使用上次
      要查找的字符串，与之对应的C-r是向上查找
    • 另外一些常见的命令
      • C-s C-w 开始递增查找操作，把光标处的单词用作查找字符串
      • C-s C-y 开始递增查找操作，把光标到行尾之间的文本当做查找的内容
      • C-s ESC-y 把删除环里面的内容当做查找的内
  • 还有很多其他类型的查找与替换，自己需要的时候找时间去研究一下，平时不
    太用的上的就不用去理会了；另外自己今后也要养成书写纯英文文献的方法，这
    样才好，使用纯英文在各个方面都可以很大程度上提高自己的工作效率，这一点
    一定要多注意才可以
  • 还存在有简写词汇的功能，这一点自己去多注意下

使用多个编辑器与缓冲区(这个是提高工作效率的一大关键)

  • 使用C-x C-f命令来打开一个新的缓冲区，使用C-x b命令来进行切换(注意这
    个命令是按下C-x之后要松开C再去按)
  • 注意，如果使用C-x C-f开错了文件，同样可以使用C-x C-v命令来进行修正
  • 支持开启多个标签，支持多个标签之间的切换，同样可以使用代码补全，非常
    好用
  • C-x b可以用来快速恢复刚才关闭掉的缓冲区
  • C-x k可以关闭不想要的缓冲区，用法同C-x b一样；删除同样可以使用命令
    kill-some-buffers来进行，这个命令会一次对每一个buffer询问过去，问你
    是否要进行删除，非常的方便
  • 遇到文件名非常奇葩时，不如是一些编号的时候，就非常不好辨认，这个时候
    就可以使用重命名buffer的方式，对buffer的重命名并不会影响到真实的文件；
    采用’rename-buffer’命令来重命名buffer
  • buffer命令冲突问题
    • 这个没有必要担心，因为会出现命名重复也是在不同的目录下，这个emacs
      会通过不同目录来进行区分，并不用太担心

关于emacs的分屏功能

  • 这个功能实在是太好用了，可以说就是开发利器，太重要了
  • 在当前光标所在的窗口使用C-x 1就会使得当前窗口最大化(关闭其他所有的窗口)，也就是变为全屏，如果想要退出分屏模式就可以考虑这个
  • 在当前光标所在的窗口使用C-x 2就会出现上下分屏；使用C-x 3就会出现左右分屏，一个窗口进行无限分隔，并没有说只能够分割为4个
  • 关闭当前的窗口可以使用C-x 0
  • 更多关于窗口的操作请看

关于emacs下repl模式的讨论

  • 能够开启repl模式一定都是在对应的编程模式下才会出现的，一般是python以及clojure
  • 开启repl一般就是与某一份正在编辑的代码连接在一起，那边编译运行，这边就会直接输出结果，所以就算隐藏了repl，再通过分屏的方式找回来也是没有关系的，因为一斤连接在一起了；但是并没有试过断开其中一个的连接会如何，比如我现在使用的是hello.clj，连接了一个repl，如果我再开一个say.clj，这个时候最好是使用同一个repl，因为不同的代码文件之间是会有关联的，如果一个编译通过了，然而另外一个没有编译过直接拿来使用就会出现很严重的问题，试一下
  • 尝试之后发现，只要打开了repl，就不需要自己再去连接了，直接编译运行就可以，全部都会在同一个repl进行输出，也就是会自动进行连接，并不需要自己操心连接问题，非常方便
  • 注意：repl如果在分屏当中使用，如果出现代码跑错的窗口，不要使用C-x k来把这个窗口关闭掉，而应该用C-x b来重开buffer，这样在一定程度上可以避免闪退的出现
