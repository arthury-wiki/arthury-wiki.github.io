Linux

环境变量

  • linux允许多用户的存在，每一个用户都有自己的专用运行环境，这个环境是
    由一组变量所定义的，注意这里是通过变量定义的，这些变量就是环境变量
  • 用户可以通过对环境变量的修改来达到对环境的要求
  • 环境变量是通过shell命令来设置的，可以被当前运行的程序所使用
  • 对于bash来说，可以通过变量来访问相应的环境变量

  • 对环境变量的基本操作

    • 环境变量其实就像是普通的变量，定义之后对其进行正常的赋值

      $ export NAME = “Example”

    • 显示：可以使用env或者printenv命令来查看所有的环境变量，使用一个
      echo命令来输出一个特定的环境变量

      $ echo $HOME

    • 使用unset来清除特定的环境变量

      $ unset NAME

    • 可以使用readonly命令来设置只读变量(设置只读属性后，其值将不可变，
      直到进程结束)

      $ export NAME=”example”
      $ readonly NAME

    • 在C语言当中也可以访问与设置环境变量

      • 传送门

  • 修改系统配置文件

    通过修改一些相关的环境定义文件来修改环境变量，与环境相关的文件有
    /etc/profile（root用户）和~/.bash_profile（普通用户，在自己的目录下，
    隐藏文件，如果没有可以新建）等。修改完毕后重新登录一次或运行命令source
    xxx就生效了。

  • 常见的环境变量

    • PATH 决定了shell将到哪些目录中寻找命令或程序
    • HOME 当前用户主目录
    • TERM 终端的类型
    • UID 当前用户的识别字，取值是由数字构成的字符
    • PWD 当前工作目录的绝对路径名，这个取值会随着当前路径的变化而变化
    • HISTIZE 历史记录数
    • LOGNAME 当前用户的登录名
    • HOSTNAME 当前主机的名称
    • SHELL 当前用户的Shell类型
    • LANGUGE 语言相关的环境变量，多语言可以修改此环境变量
    • MAIL 当前用户的邮件存放目录
    • PS1 shell命令行基本提示符，对于root用户是#，对于普通用户是$
    • PS2 附属提示符，默认是”>”

  • screen的使用

    • 介绍：可以使用screen来在服务器上跑多个任务，因为服务器上面并没有多窗口这种东西，另外多窗口其实并不好管理，然而使用screen就可以非常方便的解决这个问题了，screen可以非常方便的解决这个问题
    • 具体的实现原理：类似于开启多个终端一样，然后可以通过快捷键进入或者退出一个终端

    • 具体的操作命令

      screen -S "服务名" //开启一个新的服务
screen -r //进入上一次开启的服务的进程
screen -r -d id号 //恢复指定id号的服务进程，在开启多个服务进程的时候使用
exit //退出当前的进程，注意就算服务崩了，也要进去正常退出，对应的screen才会正常的关闭
control + a + d //离开当前screen进程

    • 出现的错误汇总

      • 使用screen -r 的时候出现 inside a screen? 这说明目前就在一个screen进程里面了，可以选择exit退出，也可以选择在这个进程里面自己把服务给跑起来

chmod命令

  • 自己写的一些脚本直接可以使用这个命令来变成可执行文件，放到\bin文件夹下，执行chmod +x 文件名，这样就可以在任意位置执行命令了，在配合alias就可以给命令写缩写了
  • 不仅支持linux的shell脚本，还支持python写的py脚本，非常的方便
