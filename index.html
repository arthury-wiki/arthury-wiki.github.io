iOS网络编程01

前言

  • 原来自己关于网络请求这一块用的都是第三方框架，其中最常用的是AFNetworking，现在我需要自己尝试自己去写这样的网络请求，然后去学习别人是如何去封装这些框架的

关于网络编程的架构设计

  • 需要能够应对不同情况下的网络环境
  • 可伸缩以及可维护

iOS提供的网络api

  • NSURL
  • GameKit
  • Bonjour
  • NSStream
  • CFNetwork
  • BSDSocket(网络框架最底层)

NSURL

  • NSURLConnection
  • NSURLSession(NSURLConnection升级版)，今后要使用的主要版本

Http简介

  • 客户端与服务器的一种数据传输协议
  • 包含的请求有：GET、POST、PUT、DELETE等等
  • GET请求主要用于向服务器获取数据，POST请求主要用于向服务器传输数据，虽然GET请求也可以做到传输数据，但是这样如果传输一些大文件，那么URL就会变得非常长，这样并不合适

NSURLSession

  • iOS7之后出现
  • 提供的几个核心功能：
    • 1.通过url将数据下载到内存中
    • 2.通过url将数据下载到文件系统中
    • 3.将数据上传到指定的url
    • 4.在后台完成上述功能
  • 优势
    • 能够在后台上传和下载
    • 能够进行暂停和恢复操作
    • 可配置性高
    • 可继承，拥有独立的私有存储
    • 改进的认证处理
    • 丰富的委托模式
    • 通过文件系统进行上传和下载
  • 工作流
    • 创建Session
    • 创建Task
    • 启动Task
    • 使用回调处理结果

NSURLSession操作实例

  • 获取指定api的数据
  • 下载远程资源
    • 创建资源的url
    • 创建configuration
      • 一般模式(default)：可以使用缓存，cookie，鉴权
      • 即时模式(ephemeral)：不使用缓存，不使用cookie，不使用鉴权
      • 后台模式(background)：在后台完成上传和下载
    • 创建DownloadTask
    • 启动task
    • 使用回调处理数据
    • 这里涉及到的代理有：下载完成时，以及下载进行中，对应的delegate是NSURLSessionDownloadDelegat

NSURLSessionConfiguration

  • 三种模式：default、background、ephemeral
  • 当一个session对象要执行上传或者下载任务的时候，就需要一个NSURLSessionConfiguration来定义上传下载的策略
  • 可以具体进行的配置：
    • 通用属性
    • Cookie策略
    • 缓存策略
    • 支持后台传输
    • 支持自定义协议
    • 设置http策略
    • 设置代理属性

NSURL的坑爹之处

  • 如果使用NSURL(string:_)的方式对URL进行初始化，就会导致初始化失败，从而返回nil，这样在解包的时候就会crash，实在是坑爹
  • 解决方案：使用String的StringstringByAddingPercentEscapesUsingEncoding:_方法进行编码，编码方式使用NSUTF8StringEncoding便可
  • 既然有编码那么对应的就有解码操作，stringByReplacingPercentEscapesUsingEncoding:_这个方法可以用来进行解码操作

  • 下面有一个非常好的例子

    NSString* string1 = @"https://www.cloudsafe.com/文件夹";  

NSString* string2 = [string1 stringByAddingPercentEscapesUsingEncoding:NSUTF8StringEncoding];  
NSString* string3 = [string2 stringByAddingPercentEscapesUsingEncoding:NSUTF8StringEncoding];  

NSString* string4 = [string2 stringByReplacingPercentEscapesUsingEncoding:NSUTF8StringEncoding];  
NSString* string5 = [string3 stringByReplacingPercentEscapesUsingEncoding:NSUTF8StringEncoding];  

NSString* string6 = [string4 stringByReplacingPercentEscapesUsingEncoding:NSUTF8StringEncoding];  
NSString* string7 = [string5 stringByReplacingPercentEscapesUsingEncoding:NSUTF8StringEncoding];  

输出结果: string5 & 6 重复编码encode  

string1:https://www.cloudsafe.com/文件夹  

string2:https://www.cloudsafe.com/%E6%96%87%E4%BB%B6%E5%A4%B9  
string3:https://www.cloudsafe.com/%25E6%2596%2587%25E4%25BB%25B6%25E5%25A4%25B9  

string4:https://www.cloudsafe.com/文件夹  
string5:https://www.cloudsafe.com/%E6%96%87%E4%BB%B6%E5%A4%B9  

string6:https://www.cloudsafe.com/文件夹  
string7:https://www.cloudsafe.com/文件夹

emacs 技巧01

emacs瞬开

  • 随着加载的配置文件的增加，emacs的启动速度越变越慢，已经慢到了让人无法忍耐的地步了，于是我去寻找了加速启动emacs的方法，于是让我在segmentfault上面找到了让emacs瞬开的技巧，下面贴上shell配置

    # alias emacs
alias emacsd='emacs --daemon'
alias e='emacsclient -t'
alias ec='emacsclient -c'

# add kill emacs function
function kill-emacs(){
    emacsclient -e "(kill-emacs)"
    emacs_pid=$( ps -C 'emacs --daemon' -o pid= )
    if [[ -n "${emacs_pid}" ]];then
        kill -9 "${emacs_pid}"
    fi
}

  • 原文地址

  • 如果配置打开终端自动启动到时候每次打开都会启动一次，输出一大串之后再显示启动失败，很麻烦，本来想加一个判断，但是试了很多方法都没有搞定，于是就移除打开终端就自动启动的设置

git使用技巧01

git本地保存

git提交的正确姿势

  • 针对各个功能点进行管理式的提交，而不是像以前一样的，每次提交都重开一行描述，这样的做法不够好，不方便查看和管理
  • 比如我正在做的功能是让顶部导航栏不覆盖tableview，那我第一次提交的信息应该是git commit -m "添加顶部导航栏不覆盖tableview"
  • 下一次我通过StroyBoard实现了另外一个版本时，就要使用git commit --amend，这样会打开上一次修改的编辑界面，然后修改为”使用SB实现”
  • git真正发挥作用的时候还要看多人协作的时候，关于这里的经验可以多多请教一下学长，看一下它们是如果使用git来做多人项目管理的，多吸收一点经验，另外抽时间把git的详细教程过一遍，现在去看会收获很多原来没有发现的东西

CocoaPods01

配置CocoaPods

  • touch Podfile
  • platform :ios, '7.0'
  • 之后的内容使用pod search xxx之后得到的信息直接拷贝便可
  • 有必要维护一个常用的第三方库配置文件
  • 好好整理一下自己star项目

移除已经安装好的库

  • 直接删除Podfile中相关配置，然后再执行pod install便可

加速CocoaPods类库的安装

  • pod install --verbose --no-repo-update
  • pod update --verbose --no-repo-update
  • 带了这些参数在执行命令的时候就不会去升级spec仓库了，这样速度会提升不少

CocoaPods更新

  • 先更新gem，sudo gem update --system
  • 再更新CocoaPods， sudo gem install cocoapods
  • 配置CocoaPods，pod setup

关于github的多账号管理

前言

  • 由于工作学习的原因需要使用多个github账号，于是如何管理这些账号就变得很重要，刚开始的时候在这里吃了不少亏，现在让我来分享一下我管理的经验

生成多个ssh公钥

  • ssh-keygen -t rsa -C "your_email@youremail.com"
  • 之后按照提示输入相关的信息便可，密码随意

添加生成的公钥

  • 这一步非常重要，千万不要忘记这一步，否则后面会很坑爹
  • ssh-add 公钥的地址(~/.ssh/xxx)
  • 可以使用ssh-add -l看已经添加的公钥，密码填写生成公钥时候填写的密码
  • 出现问题多看看这一步是不是出现了问题，也就是先把添加的公钥列出来看一下

修改配置文件config

# github
Host arthury_wiki(执行git clone等命令时候的区分)
    HostName github.com
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/xxx

配置远程仓库地址

  • 正常是git@github.com:YourAcccountName/YourRepoName.git
  • 为了区分各个账号，就需要使用配置文件中的Host来进行区分，改为git@arthury_wiki:YourAcccountName/YourRepoName.git

去网站自己抓取

  • 比如想要知道网易云音乐的搜索api，只需要去网易云音乐官网随便搜索一个关键词，然后查看url便可得到搜索的api；比如搜索月亮就会得到http://music.163.com/#/search/m/?s=月亮&type=1，这样自己就可以根据这个使用具体的工具比如Paw来测试各种参数的效果，从而得到详细信息

Mac优化清理

前言

  • 作为低配版Mac的使用者，128g的空间感觉越来越不够用了，然而新版的Mac已经无法再自行更换硬盘，实在是心累；既然这样，对于越来越小的硬盘空间，就不得不采取一系列的优化方案了

关闭Mac休眠模式

  • 打开Termal，输入sudo pmset hibernatemode 0(默认是3)
  • 然后再输入sudo rm /var/vm/sleepimage
  • 再创建一个空白的sleepimage文件sudo touch /private/var/vm/sleepimage
  • 将这个空白的文件设为只读sudo chflags uchg /private/var/vm/sleepimage
  • ok
  • 使用命令pmset -g可以参看各个属性的设置情况

禁用Mac虚拟内存

  • 禁用虚拟内存的前提是Mac的内存大小在4GB以上，否则不要轻易尝试
  • 具体原理参见在 Mac 上使用“活动监视器”
  • 在Termal中中输入sudo launchctl unload -w /System/Library/LaunchDaemons/com.apple.metadata.mds.plist
  • 禁用虚拟内存之后，再输入sudo rm /private/var/vm/swapfile*删除交换文件
  • 如果之后想要重启虚拟内存可以输入sudo launchctl load -w /System/Library/LaunchDaemons/com.apple.metadata.mds.plist
  • 现在已经不推荐这个做法了，禁用虚拟内存带来的坏处比它带来的好处更多
  • 想要临时释放不活跃的内存可以执行purg，需要Xcode支持

Xcode文件清理

  • 移除对旧设备的支持
    • 在终端输入open ~/Library/Developer/Xcode/iOS\ DeviceSupport
    • 在打开的文件夹当中删除对低版本系统的支持文件
  • 移除旧的Docsets文档
    • 在终端输入open ~/Library/Developer/Shared/Documentation/DocSets
    • 在打开的文件夹当中删除旧的Docsets文档

DaisyDisk

  • 这个简直神器，可以帮你快速分析出磁盘中各个文件夹的大小，帮你迅速锁定占据大空间的文件夹，排查那些被浪费的空间，而且软件的界面非常的优雅，实在是难能可贵
