QuartzCode01

概述

  • QuartzCode是一个矢量绘图和动画app，可以帮助你绘制图形和制作动画，并且转换成Objective-C代码或者是Swift代码，QuartzCode对于开发iOS以及OSX开发者来说非常的重要

Layer List

  • Layerlist展示了所有的layer。这些layer是根据layer的继承等级展示出来的并且通过着色的缩略图的形式展示出来。你可以通过在Layerlist中改变layer的顺序来改变layer的继承等级
  • 所有的layer都向CALayer一样运作或者它们是QuartzCore的子类。取决于它们在QuartzCode中的layer类型，这些layer的行为可能不太一样
  • 你可以通过Layerlist选择layer或者高亮它们，你同样可以通过Layerlist隐藏或者锁定它们

Animation List

  • AnimationList是管理各种动画的地方，可以在这里添加或者是删除动画

Layer Inspector

  • Layer Inspector可以让你改变layer的属性设置，不同的layer类型有着不同的属性设置

Animation TimeLine

  • Animation TimeLine控制板展示了为layer准备的动画，它可以为所有的layer展示动画，为layer准备的动画只包含动画以及被选中的layer的动画，这些设置可以通过F1，F2以及F3来进行修改

Keyframe Inspector

  • Keyframe Inspector被用来改变动画以及它的keyframe设置

Sketch01

Layer介绍

  • handler
    • 图中白色的八个小方块就是handler
    • 可以利用这8个小方块来控制layer的形状
  • 移动layer
    • 可以选择layer不放开进行拖动
    • 可以选中之后使用上下左右按键来进行移动，使用这种方法可以每次只移动1个单位
    • 如果按住shift再使用上下左右按键来移动，每次就可以移动10个单位
  • duplicate
    • 选中layer然后右键菜单，点击duplicate就可以复制出一个一模一样的layer出来
    • 按住option键然后选中layer，这样就可以直接拖出当前layer的副本了，这是一个快捷方式
  • 隐藏layer
    • 如果不需要当前的layer，又不想删除它，就可以选择隐藏layer；在左边的layerlist中选中想要隐藏的layer，然后点击眼睛图标，就可以隐藏当前的layer了
  • 锁定layer
    • 当想要当前layer锁定在画布当中不被选中时，比如作为背景的时候，就可以考虑锁定layer，选中想要锁定的layer，然后右键选择lock layer便可，这样这个layer就不能够在画布当中被选中了，但是仍然可以在左边的layerlist中进行选择
  • group layer
    • 可以将选中的layer全部合成一组，这个操作很常见，这里就不详细介绍了
    • 想要操作一组layer里面的单个layer，可以在画布当中双击这个layer，这样就可以单独操作这个被选中的layer了
  • 显示layer之间的距离
    • 如果想知道选中layer与其他layer之间的距离，那么就可以按住option键然后将鼠标移到其他的layer上面，这样就可以看到距离提示了
  • 颜色的填充
    • 颜色的填充有很多的样式，具体的效果可以自己在fill选项卡当中进行选择，有纯颜色填充，有渐变，各种各样都有，具体效果的设计自己在实战当中去把握
    • 有一个很强大的功能要注意，就是点击上图中的试管可以对屏幕中任意位置进行取色，非常强大的功能，如果看到网上有什么好的颜色自己调不出来的，就可以覅用这个功能来取色，非常的方便
  • border属性
    • 这个属性是边框的样式，可以调整边框的颜色，宽度，颜色等等；点击齿轮按钮还可以设置更加细节的东西，比如说边框是否是虚线，是否为圆角等等都是可以设置的
  • shadow属性
    • 这个是layer的阴影属性
  • 拷贝CSS样式
    • 选中layer，右键可以拷贝当前layer的CSS样式，不过生成的代码貌似很糟糕，大多数是否都是自己去写比较合适
  • Blending
    • 通过这个属性可以查看颜色混合之后的效果，虽然不知道有什么卵用
    • 将一个layer的blending属性修改成非Normal之后就可以将这个layer覆盖到其他的layer上面查看混合的效果了

      画布

  • 移动画布
    • 按住空格键在点击画布的任意区域，就可以拖动画布了
  • show rulers
    • 可以通过tool bar中的views按钮选择这个功能，这样会在画布周围配上尺子来辅助布局
  • 放大与缩小画布
    • 除了可以使用tool bar上面的放大镜来进行缩放之外，还可以按住z键，然后点击任意你想要放大的区域；按住option+z，然后点击任意你想要缩小的区域
    • 还可以使用command +/-来放大与缩小；使用command 0来回到100%的比例

简单了解万维网

  • 万维网是一个由许多互相链接的超文本页面(也就是网页)组成的系统。
  • 网页使用网址(URL)进行定位，并链接彼此
  • 网页使用http协议传输

爬网页的流程

  • 1.找到包含我们需要的信息的URL列表
  • 2.通过http协议把页面下载回来
  • 3.从页面的html中解析出需要的信息
  • 4.找到更多的url，回到

选取一个Starter URL

  • 我们以爬去豆瓣电影作为入门教程
  • 既然我们要爬所有的电影，首先我们需要抓一个电影列表(也就是介绍电影的URL列表)，一个好的列表应该：
    • 包含足够多电影的URL
    • 通过翻页，可以遍历所有电影
    • 一个按照更新时间排序的列表，可以更快抓到最新更新的电影(或者是按照经典进行排序的电影，根据自己的需求来决定)

创建一个项目

  • 在pyspider的dashboard中点击”Create”按钮，命名项目为”tutorial_douban_movie”

  • 替换on_start函数的self.crawl的URL：

    @every(minutes=24 * 60)
def  on_start(self):
    self.crawl('http://movie.douban.com/tag/', callback=self.index_page)

  • self.crawl告诉pyspider抓取指定页面，然后使用callback函数对结果进行解析；每一个抓取都要配置一个指定的callback来进行解析

  • @every修饰器，表示on_start每天会执行一次，这样就能抓到最新的电影了
  • 点击run，就可以看到follows上面有一个红色的1，点击follows面板，再点击绿色的播放按钮便可

Tag列表页

  • 在tag列表页，我们需要提取所有的电影列表页的URL；目前sample handler一斤提取了非常多大的URL，所有，一种可行的提取列表页URL的方法就是使用正则表达式进行过滤：

    import re
...

    @config(age=10 * 24 * 60 * 60)
    def index_page(self, response):
        for each in response.doc('a[href^="http"]').items():
            if re.match("http://movie.douban.com/tag/\w+", each.attr.href, re.U):
                self.crawl(each.attr.href, callback=self.list_page)

  • 由于电影列表页和tag列表页不太一样，因此在这里新建以另一个callback作为self.list_page

  • @config(age=10 24 60 * 60)在这表示我们认为10天内页面有效，不会再次进行更新抓取(这里age表示多久之后需要重新抓取进行分析)
  • 由于pyspider是纯python环境，你可以使用python强大的内置库，或者你熟悉的第三方库对页面进行解析

环境

递归

  • 在自己的工作步骤中，包含着调用这个规则本身的需要；使用递归的思想可以简单的描述深度嵌套的情况；到现在我都有点害怕的=递归这种东西，总觉得很难去理解，自己调用自己，我潜意识里面认为自己调用自己不会有改变，其实不然，递归调用在每一次调用之后都会改变输入的参数，直到不再调用自身可以确切的得到答案位置
  • 递归过程与递归计算过程
    • 递归过程是指在计算过程中有出现自己调用自己；而递归计算的过程指的是上一步的计算执行依赖于下一步计算的结果，就是有使用都推迟计算
  • 递归计算过程与迭代计算过程的区别
    • 迭代计算过程每个人都需要保存纸条(每个人手上纸条的数量还不一样)，迭代计算结束只是在传递纸条
    • 迭代计算过程能用明确的变量，保存计算机过程中的状态；而递归计算过程除了保存形式变量中的信息，计算机还要保存一些不可见的信息，也就是都有哪些过程是被推迟计算的信息
  • 怎么写递归计算过程的程序？怎么写迭代计算过程的程序？如何进行选择？
    • 递归计算过程实际上有点类似于对计算的拆解
      • 1.首先要明确递归的边界条件，也就是递归到了什么地方会有确切的解，而不是调用自己
      • 2.要明确调用自己的意义，调用自己一定是为了能够达到边界条件而努力，利用调用自己得出来的结果组合出需要的答案，要明白自己最终一定会得到结果；就好像下面的例子中调用自己相加是根据斐波那契数列的定义进行组合的
      • 3.调用自身之后会返回什么？首先参数会减小，其次返回的是整个过程本身，可以画成树状图来进行分析，一目了然

  • 递归目前有两种形式

    • 1.第一种是斐波那契数列那样递归是表达自己的定义
    • 2.第二种是像求阶乘那样对问题进行分解

    • (Define (FIB N)
    (if (< N 2)
        N
        (+ (FIB (- N 1))
            (FIB (- N 2)))))

  • 递归的实质就是模拟一个过程的完成，然后通过缩小参数来控制自动演算完剩下的所有过程

语法的糖衣

  • 比如要定义一个平方运算的实现，规规矩矩的写需要很多行代码，但是通过语法糖可以采用简单的写法却实现相同的效果，说白了就是一种简化的写法

    //定义一个平方运算
//方案一：原始的写法，符合Define的标准定义模式，其中Lambad用于构建一个过程，square与过程绑定在一起
(Define square (Lambad (x) (*x x)))

//方案二：语法糖的写法，本质是上面的写法，这是一个命名绑定的过程；上面一个方案其实也可以认为是语法糖的展开
(Define (square x) (* x x))

函数与过程

  • 函数是描述一件事情的特征，过程是描述如何去做这件事；函数说平方根的定义是这样的，而过程是平方根是要怎么样才可以求出来；编写的程序指导着过程的进行，通过语言来形式化的描述这一个过程

块结构

  • 构建复杂系统是要解决的核心问题；构建计算机程序是一个抽象的过程，不像其他的领域，存在种种约束

黑盒抽象

  • 可以利用黑箱子去祝贺出更大的黑箱子；箱子的输入与输出不一定就是数字，还可以是一个过程，输入是一个过程，输出也可以是一个过程；就好比求函数不动点的方法，用来求平方根只是它的一种特例用法，它还可以用在很多其他的方面上
  • 黑盒抽象的精髓在于你不需要知道利用的东西到底是内建的支持还是别人自定义的过程

组合式

  • 组合式由操作符和运算对象组成
  • 运算对象本身也可以是一个组合式
    0 lisp当中采用前缀表达式，操作符放在前面，运算对象放在后面；被括号起来就代表这是一个组合式，意味着要将运算符作用于运算对象
  • 前缀表达式的使用规则
    • 括号的最左边一定是一个操作符，可以是+-，也可以是define；右边是运算对象，对于+-而言，运算对象只能有两个，当然运算对象本身可以是一个组合式，因此可以嵌套；define用法相同，可以用来绑定符号与数字，也可以用来绑定符号与运算过程，使用方法同+—；但是define在绑定过程上有一个语法糖的存在，把自己会传入的参数提取出来，放在一个括号里面，后面再跟着过程主体

  • 使用S表达式最大的优点就是可以不需要去关心所谓的运算顺序，因为括号的存在使得根本就不需要去在意这种规则，无论从哪一个开始计算其结果都是一样的

    //绑定数字
(Define x 2)
//绑定过程
(Define (square x) (* x x))
//Define后面一旦遇到一个括号，就会进入语法糖模式
(Define A (* 5 5)) //绑定一个数字
(Define (D) (* 5 5)) //绑定一个过程
//上面两个结果是一样的，但是解释器的执行是不一样的
//在解释器输入A会返回25；输入(A)，会提示错误，因为A不是一个运算符，一个操作过程
//在解释器输入(D)会执行D过程，然后返回25；直接输入D会提示错误

  • define可以绑定不止一个过程，在第一个括号之后，后面的都是要尽心绑定的一系列过程，绑定完执行表达式的时候就是从绑定的第一个过程开始执行

    (Define (sqrt x)
  (Define (improve guess)
    (average guess (/ x guess)))
  (Define (good-enough? guess)
    (< (abs (- (square guess) x))
        .001))
  (Define (try guess)
    (if (good-enough? guess)
        guess
        (try (improve guess))))
  (try 1))

代换模型

  • 对于基础过程的定义
    • 没有必要太过于纠结一些基础过程的实现，就好像+-如何实现一样，去深入研究这些队研究无益

公共模式

  • 给过程命名，然后可以将过程作为参数尽心传递，过程参数会在相应位置展开，就像是将一大段代码打包作为参数进行传递一样，这一块还需要好好的研究
  • 当很多的代码共享一种抽象的基础模型，就可以抽象出一种公共模式；define中的参数，可以当做数字来使用，那就是数字参数，也可以当做过程来使用，那就是过程参数，这是非常神奇的一个地方
  • 使用公共模式可以抽象，使得大多数的运算变得易于维护，只需要修改公共模式的实现，就可以做到修改所有使用公共模式来实现的运算
  • 把过程作为参数进行传递可以在被调用的位置直接展开，非常简单

Lambad

  • 生产匿名过程
    • 可以使用Lambad关键字生成一个匿名的过程，由于是匿名的过程，所以写完之后一定就要直接调用了，因为是匿名的，所以无法获得引用，如果不在写完之后直接调用，就会失去意义

      let colorSpace = CGColorSpaceCreateDeviceRGB()

      //4 - set up the color stops
      let colorLocations:[CGFloat] = [0.0, 1.0]

      //5 - create the gradient
      let gradient = CGGradientCreateWithColors(colorSpace, 
                                                colors, 
                                                colorLocations)

      //6 - draw the gradient
      var startPoint = CGPoint.zeroPoint
      var endPoint = CGPoint(x:0, y:self.bounds.height)
      CGContextDrawLinearGradient(context, 
                                  gradient, 
                                  startPoint, 
                                  endPoint, 
                                  0)
    }
}

  • 上面的代码做了以下这些事：

    • 使用@IBInspectable标记了startColor与endColor两个属性，这样就可以在StroyBoard中调整了
    • CG的绘图函数需要指导它们需要在哪里绘图的上下文，所以你需要使用UIKit的方法UIGraphicsGetCurrentContext()来获取当前的上下文，这也是drawRect(_:)方法会载入的上下文(也就是在drawRect(_:)中调用CG的绘图函数并不会使得自动载入当前的绘图上下文，只有使用UIKit封装好的绘图函数才会自动载入当前的上下文，比如UIBezierPath，这一点非常重要，一定要记住了)
    • 所有的绘图上下文都有一个color space，这个可能是CMYK也可能是grayscale，但是这里你使用的是RGB color space
    • 这些起止颜色描述颜色渐变从哪里开始改变。在这个例子当中，你只有两种颜色，红色渐变成绿色，但是你也可以拥有三种起止颜色的数组，从红色将变成蓝色再到绿色。这些色标在0到1之间变化，0.33三分之一的梯度
    • 创建一个完全的梯度，定义color space(想要采用的描述颜色的方式，有三种范式可以选着)，colors(用来渐变的颜色有哪些)以及color stops(颜色渐变开始与结束地方).
    • 最后，你绘制了这个梯度。CGContextDrawLinearGradient()接受一下参数：
      • CGContext是需要载入的上下文
      • CGGradient包含了color space, colors and stops信息
      • 开始的点
      • 结束的点
      • 拓展梯度的额外选项
  • 这样梯度会填充满整个画布

  var y:CGFloat = CGFloat(graphPoint) / 
          CGFloat(maxValue) * graphHeight
  y = graphHeight + topBorder - y // Flip the graph
  return y
}

在Core Graphics的背后

  • 每一个UIView有一个绘图上下文，在视图被转换到设备的之前，所有的绘图工作都会完成渲染
  • iOS通过调用drawRect(_:)来更新绘图上下文，无论何时视图需要被更新都要调用这个方法，这个方法会在一些情况被调用：
    • 当视图第一次载入的时候
    • 顶部的其他视图被移走的时候
    • 当视图的hidden属性被改变的时候
    • 你可以通过直接调用setNeedsDisplay()和setNeedsDisplayInRect()方法来调用这个方法
  • 注意：所有在drawRect(_:)中进行的绘图工作都载入到视图的绘图上下文当中。如果你开始在drawRect(_:)之外进行绘图，就像你会在最后一部分教程中做的一样，你将会不得不创建你自己的绘图上下文
  • 你还没有使用Core Graphic，因为UIKit已经封装了许多的Core Graphic函数。UIBesizerPath就是对CGMutablePath的封装，这是对底层的Core Graphics API进行封装
  • 注意：绝不要直接调用drawRect(_:)方法，如果你的视图没有被更新，那么就使用setNeedsDisplay()；setNeedsDisplay()自身并没有调用drawRect(_:)，但是它会将视图标记为’dirty’，会在下一次屏幕的更新循环当中使用drawRect(_:)进行重绘，就算你在同一个方法当中调用setNeedsDisplay()五次，实际上也只会调用drawRect(_:)一次

点和像素

  • 追溯到最开始的iPhone，点和像素占据了相同的空间并且是相同的大小，使得它们完全就是相同的东西。当retina屏幕的iPhone出现之后，突然像素的数量变成了点的四倍
  • 相似的，iPhone6 Plus又一次增加像素的数量
  • 注意：接下来的是概念，实际设备上的像素可能稍有不同。注意学习使用paint code来辅助自己快速绘图，参考资料
    -这里有12x12的网格，点有灰色和白色。第一个iPad2中点和像素是一样对应的；第二个是2x的retina显示屏，这里4个像素组成一个点；第三个是一个3x的retina屏幕，这里9个像素组成一个点
  • 你绘制的线有3个点的高度，线条是从轨迹的中心开始画的，所以轨迹的两边分别都是1.5个点
  • 下面这张图片展示3个点的高度在各个设备中被绘制出来的情况。你可以看到iPad2和iPhone6 Plus都有划线穿过半个像素，这是无法被完成的。所以，iOS就会在这一半被填充的像素使用一种模糊不清的处理方式(anti-aliases)
  • 实际上，iPhone6 Plus有着非常多的像素，所以模糊影像看起来就不是那么的明显了，即便如此你应该确定你的app在不同设备上的表现情况。但是如果你是为非retina显示屏的设备比如iPad2或者iPad mini做开发，你应该极力避免出现anti-aliases的情况，因为像素点不多，所以模糊的效果非常的明显
  • 如果你有奇怪厚度的直线，你需要在给它们定位的时候+0.5或者是-0.5来防止出现anti-aliasing的情况。如果上面的图表，你将会看到在iPad2上面出现的一个点被分成两半的情况，在这种情况下就应该要把点往上移动半个像素，在iPhone 6上面，不需要做移动，在iPhone Plus上面，同样移动半个像素

x:bounds.width/2 - plusWidth/2,

    import UIKit

@IBDesignable
class PushButtonView: UIButton {

  @IBInspectable var fillColor: UIColor = UIColor.greenColor()
  @IBInspectable var isAddButton: Bool = true

  override func drawRect(rect: CGRect) {

    var path = UIBezierPath(ovalInRect: rect)
    fillColor.setFill()
    path.fill()

    //set up the width and height variables
    //for the horizontal stroke
    let plusHeight: CGFloat = 3.0
    let plusWidth: CGFloat = min(bounds.width, bounds.height) * 0.6     

    //create the path
    var plusPath = UIBezierPath()

    //set the path's line width to the height of the stroke
    plusPath.lineWidth = plusHeight

    //move the initial point of the path
    //to the start of the horizontal stroke
    plusPath.moveToPoint(CGPoint(
      x:bounds.width/2 - plusWidth/2 + 0.5,
      y:bounds.height/2 + 0.5))

    //add a point to the path at the end of the stroke
    plusPath.addLineToPoint(CGPoint(
      x:bounds.width/2 + plusWidth/2 + 0.5,
      y:bounds.height/2 + 0.5))

    //Vertical Line
    if isAddButton {
      //move to the start of the vertical stroke
      plusPath.moveToPoint(CGPoint(
        x:bounds.width/2 + 0.5,
        y:bounds.height/2 - plusWidth/2 + 0.5))

      //add the end point to the vertical stroke
      plusPath.addLineToPoint(CGPoint(
        x:bounds.width/2 + 0.5,
        y:bounds.height/2 + plusWidth/2 + 0.5))
    }

    //set the stroke color
    UIColor.whiteColor().setStroke()

    //draw the stroke
    plusPath.stroke()

  }

}

即兴的数学课

  • 我们短暂的插入这个教程，回顾一下高中时候上的数学课。就像道格拉斯说的–不要惊慌！:]
  • 载入上下文的绘图代码是在单位圆的基础上进行的，单位圆就是半径为1的圆。
  • 上图中红色的部分你的弧将会从哪里开始与结束，顺时针方向绘图。你将会从3π/4的地方开始画弧，画到π/4结束。
  • 弧度常常在代码中代替度数，使用弧度来思考是很有必要的，这样你就不要每一次都转换为度数了。之后你需要指出弧线的长度，这个时候度数就会起到作用。
  • 一个弧在单位圆中的长度就像是角度在弧度中的测量一样。举个栗子，看看上面的图表，弧线的长度从 0º到90º是π/2。为了计算出弧线的长度，需要将在单位圆中的弧长等比例放大到真实圆的半径当中。

  • 为了计算红色弧长的长度，我们需要进行一下的计算：

    2π – end of arrow (3π/4) + point of arrow (π/4) = 3π/2

  • 度数是

    360º – 135º + 45º = 270º

画出弧线的轮廓

  • 当用户标记了他们喝的水之后，计数器的轮廓就会展示出他们喝的倍数占了8杯的比例
  • 这个轮廓由两个弧线组成，外弧线和内弧线，并且有两条线连接它们。

  • 在CounterView.swift里面，将这些代码添加到drawRect(_:)里面：

    //Draw the outline

//1 - first calculate the difference between the two angles
//ensuring it is positive
let angleDifference: CGFloat = 2 * π - startAngle + endAngle

//then calculate the arc for each single glass
let arcLengthPerGlass = angleDifference / CGFloat(NoOfGlasses)

//then multiply out by the actual glasses drunk
let outlineEndAngle = arcLengthPerGlass * CGFloat(counter) + startAngle

//2 - draw the outer arc
var outlinePath = UIBezierPath(arcCenter: center, 
                                  radius: bounds.width/2 - 2.5,
                              startAngle: startAngle, 
                                endAngle: outlineEndAngle,
                               clockwise: true)

//3 - draw the inner arc
outlinePath.addArcWithCenter(center, 
                     radius: bounds.width/2 - arcWidth + 2.5,
                 startAngle: outlineEndAngle, 
                   endAngle: startAngle, 
                  clockwise: false)

//4 - close the path
outlinePath.closePath()

outlineColor.setStroke()
outlinePath.lineWidth = 5.0
outlinePath.stroke()
