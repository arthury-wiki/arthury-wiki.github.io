Linux

环境变量

  • linux允许多用户的存在，每一个用户都有自己的专用运行环境，这个环境是
    由一组变量所定义的，注意这里是通过变量定义的，这些变量就是环境变量
  • 用户可以通过对环境变量的修改来达到对环境的要求
  • 环境变量是通过shell命令来设置的，可以被当前运行的程序所使用
  • 对于bash来说，可以通过变量来访问相应的环境变量

  • 对环境变量的基本操作

    • 环境变量其实就像是普通的变量，定义之后对其进行正常的赋值

      $ export NAME = “Example”

    • 显示：可以使用env或者printenv命令来查看所有的环境变量，使用一个
      echo命令来输出一个特定的环境变量

      $ echo $HOME

    • 使用unset来清除特定的环境变量

      $ unset NAME

    • 可以使用readonly命令来设置只读变量(设置只读属性后，其值将不可变，
      直到进程结束)

      $ export NAME=”example”
      $ readonly NAME

    • 在C语言当中也可以访问与设置环境变量

      • 传送门

  • 修改系统配置文件

    通过修改一些相关的环境定义文件来修改环境变量，与环境相关的文件有
    /etc/profile（root用户）和~/.bash_profile（普通用户，在自己的目录下，
    隐藏文件，如果没有可以新建）等。修改完毕后重新登录一次或运行命令source
    xxx就生效了。

  • 常见的环境变量

    • PATH 决定了shell将到哪些目录中寻找命令或程序
    • HOME 当前用户主目录
    • TERM 终端的类型
    • UID 当前用户的识别字，取值是由数字构成的字符
    • PWD 当前工作目录的绝对路径名，这个取值会随着当前路径的变化而变化
    • HISTIZE 历史记录数
    • LOGNAME 当前用户的登录名
    • HOSTNAME 当前主机的名称
    • SHELL 当前用户的Shell类型
    • LANGUGE 语言相关的环境变量，多语言可以修改此环境变量
    • MAIL 当前用户的邮件存放目录
    • PS1 shell命令行基本提示符，对于root用户是#，对于普通用户是$
    • PS2 附属提示符，默认是”>”

  • 标准输入与输出

    • Linux Shell 文件描述符 及 stdin stdout stderr 重定向

  • screen的使用

    • 介绍：可以使用screen来在服务器上跑多个任务，因为服务器上面并没有多窗口这种东西，另外多窗口其实并不好管理，然而使用screen就可以非常方便的解决这个问题了，screen可以非常方便的解决这个问题
    • 具体的实现原理：类似于开启多个终端一样，然后可以通过快捷键进入或者退出一个终端

    • 具体的操作命令

      screen -S "服务名" //开启一个新的服务
screen -r //进入上一次开启的服务的进程
screen -r -d id号 //恢复指定id号的服务进程，在开启多个服务进程的时候使用
exit //退出当前的进程，注意就算服务崩了，也要进去正常退出，对应的screen才会正常的关闭
control + a + d //离开当前screen进程

    • 出现的错误汇总

      • 使用screen -r 的时候出现 inside a screen? 这说明目前就在一个screen进程里面了，可以选择exit退出，也可以选择在这个进程里面自己把服务给跑起来

chmod命令

  • 自己写的一些脚本直接可以使用这个命令来变成可执行文件，放到\bin文件夹下，执行chmod +x 文件名，这样就可以在任意位置执行命令了，在配合alias就可以给命令写缩写了
  • 不仅支持linux的shell脚本，还支持python写的py脚本，非常的方便

常用快捷键

  • Meta键的映射，目前键盘上并没有找到
  • C-a 移动到一行的开头；C-e 移动到一行的结尾
  • M-a 移动到句首；移动到句末
  • ESC } 下一段落
  • ESC { 上一段落
  • C-x C-s(save-buffer)，保存缓冲区文件
    • 出现问题之后可以采用C-s(停止接受输入) C-q(激活控制流)
    • 也可以使用enable-flow-control命令来进行解决
  • ESC x 可以用来输入没有键盘快捷键的命令，输入之后按下return
  • C-x C-w(write-file)，重命名并保存文件，不命名则使用原来的名字
    • 这个命令同样可以用来创建一个不可修改文件的副本，重命名保存到不同的路径便可，这样就可以有一个可以修改的私有版本了
  • C-x C-c，退出Emacs
  • 在文件末尾插入文件的快捷键
    • 先按下esc，松开之后再按下>(就是shift+.)，然后再输入C+x，稍等一下出现提示之后，再按下i键便可，会自动跳到文件末尾插入
    • 另外esc + >也可以单独用来作为跳到文件末尾的快捷键
    • esc + < 可以调到文件的开头
  • C-x C-f 打开文件
  • C-x C-v 重新选择要打开的文件
  • C-u 8 C-v 向下滚动8行，当然数字是可以改变的
  • C-g 可以取消不想要执行的命令，注意是g
  • C-h可以调出帮助文档，当然需要相应的参数
    • t 教程
    • k 按键释义
    • f 函数释义
    • i info文档
  • C-x i 在当前为止插入文件
  • C-q C-l 在当前为止插入分页符
  • C-x ] 移动到下一个分页符所在位置
  • C-x [ 移动到下一个分页符所在位置
  • C-/ 撤销操作，undo，类似的操作还有C-y，还有ESC y可以恢复删除环里面所有的文本
  • C-r redo，也就是取消撤销的操作
  • C-l 重新绘制屏幕，当屏幕变得有点混乱之后就可以使用这个来整理一下屏幕
  • 删除操作
    • C-d 删除一个字符
    • ESC-d 删除的是整个单词(具体的效果要自己在实战当中进行总结)
    • C-k 可以做到删除一整行的作用，注意使用的时候一定要把光标移动到开头，
      删除是从光标之后开始的
    • ESC-k 删除光标之后的句子直到段尾
    • C-x DEL 删除光标之前的句子直到段首
    • C-w 删除文本块，就是从光标开始，要设置一个文本文本标记(Mark set)，
      mark set与当前的光标所包围的区域就是文本块，这时候就是可以使用C-w进
      行删除，使用C-space来设置文本标记点
      • 可以互换两个标记的地方，使用C-x C-x组合键来实现
      • C-w其实是剪切，剪切下来的文本在适当的地方使用C-y来进行黏贴
      • 大范围的标记
        • ESC h 自动标记当前光标所在的段落
        • C-x h 自动标记整个缓冲区的内容
        • 如果已经有使用页面的分页符，那么可以使用C-x C-p来快速标
          记整个页面
    • ESC-C-k 效果是删除从行首(不是整个文件的首部)开始到光标所在的位置
      (这一按下esc之后还有一条横线，不要忘记这个)
    • 被删除之后的数据会进入到删除环里面，等着随时被恢复，实际上删除环里
      面可以容纳30次左右的删除，当然也可以自己通过命令来扩充容量，但是一般不会有这样的必要
  • 复制与删除
    • 使用文本标记辅助自己选中想要复制的文本块，然后按下ESC-w进行复制，

  • 没有键盘快捷键的命令(输入命令的时候同样可以使用Tab进行补全)
    • auto-fill-mode 开启/关闭自动换行模式，看一下状态栏上面有没有fill，

    • goto-line n 跳转到第n行

  • 简化编辑的小技巧
    • C-t 可以交换前后两个字符的位置，一般在打英文的时候会使用到，现在想想纯英文的世界实在是太美好了，不用去在意输入法的问题

  • 撤销所有修改，返回最后一次保存的状态
    • revert-buffer 这将会让文件直接返回到最后一次存盘的状态，是恢复当前的文件

自动备份的文件

  • Emacs会在第一次保存某一个文件的时候创建一个备份，就是平时看到的文件名后面加一个~，但是并不能直接从备份文件中进行恢复的命令
  • 使用备份文件覆盖当前文件的操作
    • 直接对~文件进行编辑，然后使用C-x C-w 文件名 RETURN来进行覆盖
    • Emacs支持保存多份的备份，每一次存盘之前都会有一个备份，可以无限备份下去，但是最后是备份最近的几次就好，这就是版本控制模式(VC mode)

出现了非正常退出Emacs的时候怎么办？

  • Emacs有一个auto-save的功能，其实就是自动保存的功能，这个会生成一个#
    文件名#的文件，这个文件在Emacs非正常退出之后就会出现，就是自动保存的文
    件内容，如果出现了非正常的退出就可以从这个文件里面恢复
  • 使用recover-this-file命令可以从自动保存的文件里面进行恢复，非常强大
  • 可以使用aoto-save-interval命令来修改自动保存的间隔，是以敲击键盘达
    到多少次来作为判断的标准
  • 如果对Emacs文件进行了大量的文件删除工作就会就不会进行自动保存，此时
    要进行一次存盘才会重新开始自动保存；或者使用ESC 1 ESC x auto-save RETURN(这里是数字1)来强制自动保存

python整体的学习架构

语言特性

  • python属于动态语言
    • python不需要知道数据类型，所以不用写类型声明，非常的爽，直接想要使用什么就去用什么便可
    • 虽然说python使用变量不需要声明，但是一个变量(标识符)一定要事先经过赋值才可以进行使用，否则就会出现NameError

  • python属于强类型语言

    • 变量类型一旦确定，就不会出现进行自动转换，比如int自动转换成float并不会进行，也就是所谓的隐式转换，那么在弱类型语言里面int类型的变量什么时候会隐式转换成float

    • 隐式转换(在C语言中)

      • 不同类型的数据在运算时计算机会先自动将其转换成同一类型之后再进行计算，主要是char->int->double<-float，这几个类型在几乎在所有类型的语言里面都是存在的

      • 隐式转换并不会改变变量的数据类型，变量的数据类型一旦确定下来，如果是在C语言当中，是没有办法再进行修改的，但是在python当中仍然是可以进行修改，因为python是解释运行的，每遇到一条代码命令，就进行编译，对于同一个变量，进行多次不同类型数据的赋值，是会被解释成不同的数据类型

        a = 1
        a = “c”
        a = 3.7

      • 如上代码最后a的类型是float

      • 如此看来隐式转换只是针对于变量里面的数据，而不是这个变量，所以改变类型的只是数据，而不是变量本身；这样看来无论是显示(强制)转换与隐式(自动)转换都是针对于数据而言，而不是针对于变量而言

列表与数组

  • 列表看起来就像是打了鸡血的数组，因为比数组要强大
    • 列表可以通过方法进行动态伸缩
    • 列表本身同样可以作为列表的元素
    • 列表允许混合数据
    • 往一个列表里面插入数据有两种方法，一种是使用insert方法，另一种是重新创建列表；无论是哪一种方法都会有适应的场景
  • 列表分别有可变与不可变的两种情况，一种是使用[]来创建，这一种是可变的；另一种是用()来创建，这一种是不可变的，也叫做元组(tuple)，怎么与swift这么像，注意元组是不可变的列表

python当中的转义字符

  • 无论是单引号还是双引号，都可以用来创建一个字符串，只要注意不可以混合使用便可
  • 同样与其他的语言一样，python也可以使用\来进行转义

  • 对一个函数进行增强，并不需要再写一个全新的函数，这样会导致选择过多，比较合理的解决方案是通过提供参数来，通过参数来控制函数的行为，这样才是比较理想的
  • 一点一点为用户考虑的设计想法实在是太棒了，让我学到了设计一个函数api如何才能够周全的考虑的问题，通过多样的参数与缺省值来控制函数的行为，而不是一点小修改就去设计一个新的api，这样会导致有太多的api，非常的糟糕

对于缺省值的理解

  • 比如一个函数要求提供很多个接口，如果有的参数并不需要传入进来，每个都去写没有又是一件非常麻烦的事情，这样可以通过缺省值来解决这个烦恼，有了缺省值，有的参数没有写，那就会自动使用这个缺省值

python的异常捕获(异常处理机制)

  • 在运行的时候出现了异常可以直接捕获到，然后对这个异常进行处理，python的异常处理非常的强大，允许错误发生，然后给你一个机会来恢复，太酷了，一旦发现了异常，可以直接进行修复之后再运行
  • try与except语句，作为处理异常的机制非常的强大
try:
    #你想要执行的功能代码
except:
    #try里面的代码发生了错误之后就会执行这里面的代码
    pass #可以用这个pass命令直接无视掉try里面的代码
  • try语句非常的使用，不仅可以让代码更加容易的进行阅读，而且出问题时更容易修正
  • except还可以指定捕捉的错误类型，也就是只在特定的错误发生的时候才会去执行except下面的代码
try:
    data = open('sketch.text')
    for each_line in data:
        #if not each_line.find(':') == -1:

except IOError:
    print('The data file is missing!')

如何理解异步模型

异步编程通常是通过fire and forget方式实现，发射事件后即忘记，做别的事情了，无需立即等待刚才发射的响应结果了。(发射事件的地方称为生产者，而将在另外一个地方响应事件的处理者称为消费者)

  • 也就是说事件一旦被发射出去之后就可以执行下一个新的任务了，发射期间是不能执行其他任务的，等到响应的结果回来之后就可以有消费者执行消费的任务，切换过来
  • 放到自己做app的经验当中来比喻就是，进行了一个下拉刷新的操作，这个时候发生了请求数据的事件，接下来仍然可以对app进行操作，等到数据回来之后数据就会被刷新，刷新数据的时候是不可以进行操作的；当然iOS的app实际并不是这样运作的，里面有多线程运作的机制
