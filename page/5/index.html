Masonry01

可以添加三个autolayout约束的函数

- (NSArray *)mas_makeConstraints:(void(^)(MASConstraintMaker *make))block;
- (NSArray *)mas_updateConstraints:(void(^)(MASConstraintMaker *make))block;
- (NSArray *)mas_remakeConstraints:(void(^)(MASConstraintMaker *make))block;
  • mas_makeConstraints 只负责新增约束 Autolayout不能同时存在两条针对于同一对象的约束 否则会报错
  • mas_updateConstraints 针对上面的情况 会更新在block中出现的约束 不会导致出现两个相同约束的情况
  • mas_remakeConstraints 则会清除之前的所有约束 仅保留最新的约束

注意事项

  • 在autolayout之前一定要先将view添加到superview上，否则会报错

布局01

最傻的布局方式

  • 通过设置frame进行布局，这是最傻的布局方式，为什么说最傻？

    • 第一：需要为不同的尺寸去设置一个frame值

    • 第二：一旦视图载入之后布局定下来，之后就算计算frame相关联的属性值发生了改变，frame也不会被重新计算，布局也不会改变，举个栗子

      UIView *testView = [[UIView alloc] initWithFrame:CGRectMake((self.view.frame.size.width-100)/2,(self.view.frame.size.height-100)/2, 100, 100 )];

    • 这里虽然计算testView位置的时候关联了self.view.frame，想着这样之后就算视图翻身了翻转变化也没有关系，testView会根据self.view的变化进行重新布局；NO!NO!NO!太天真了，根本就不会发生变化，因为这个值并不会因为关联数据的变换而重新进行计算，一旦尺寸确定下来就不会再发生变化了，切记这一点；所以非常的不方便，特写是那种需要实时重新计算布局情况，使用这个肯定不行

使用AutoLayout

使用Nib的隐藏优势

  • 自动具有实时计算frame的特性，只用读取了nib文件，就算是直接对frame进行设置，一旦关联值发生了变化，frame值就会被重新计算，这是自带的优势，可以省去很多写代码的麻烦，因此我需要进行取舍才可以，自己应该把精力发在哪一块代码上面，是放在布局上面，还是把时间画在动画效果的制作上面；实际上要懂得把各个有点结合在一起使用才可以，确实可视化的布局可以节省很多时间不用去写那些无聊的布局代码，但是使用代码可以让自己学到很多东西，对于要研究底层的我，知道这些还是很有必要的

NSTimer(翻译)

什么是NSTimer？

  • NSTimer类可以被用来创建timer对象，简单的设置一个定时器。我确信你看到过许多的iOS应用有类似的功能。曾经想要知道它们是如何工作的？想要知道NSTimer是如何工作的？好，它等待经过一段特定的时间之后会被触发。触发它会给你的app发送一个特定的信息，举个栗子，你可以告诉NSTimer对象发送一个信息给window，让其在一段时间之后更新，或者是每一秒更新一个label
  • 那么现在，你想要指导如何在Swift当中使用NSTimer，不是吗？我将会在这篇教程里面展示给你看。

Start NSTimer

  • 在IBAction输出的startButton我们需要添加：

    timer = NSTimer.scheduledTimerWithTimeInerval(TIME_INCREMENT, target:self, selector: Selector("FUNCTION"), userInfo: nil, repeats: BOOL)

  • 这行代码将会生成一个NSTimer的实例赋值给这个timer变量，在这个方法里里面有一些参数；TIME_INCREMENT，FUNCTION，BOOL。下面将对这些参数进行介绍
    || 参数 || 用途 ||
    || TIME_INCREMENT || 触发定时器的秒数，如果秒数比0.0还要少或者相等，这个方法将会选择和非负数0.1毫秒来替代，我们将会将这个参数设置为1 ||
    || FUNCTION || 当定时结束之后会触发的函数，我们将会调用方法updateCounter ||
    || BOOL || 如果是YES，这个触发器就会一直重复直到被销毁；如果是NO，这个定时器在执行之后就会被销毁，我们将会设置这个值为YES ||

  • 所以现在，这一行代码看起来应该是这样的

    timer = NSTimer.scheduledTimerWithTimeInterval(1, target:self, selector: Selector("updateCounter"), userInfo: nil, repeats: true)

  • 下面我们需要做的局势创建一个updateCounter以及更新这个counter变量为1。就像我们在viewDidLoad方法中定义的那样，然而我们需要添加++给这个counter，这样这个值在每一次函数被调用之后这个值就会被加1.我们已经告诉NSTimer.scheduledTimerWithTimeInterval每间隔1秒调用updateCounter，就像这样：

    func updateCounter() {
    countingLabel.text = String(counter++)
}

  • 运行你的app，下面就是见证奇迹的时刻

  • 这个类使用了一个方法invalidate()，这个方法可以完美的来停止与暂停你的SNTimer。在pause button的IBAction里面我可以需要加入这一行代码：

OpenGL ES01

前言

  • OpenGL ES作为大部分游戏引擎的底层框架，它直接面向的是GPU，因此拥有非常强劲的性能，作为需要进行游戏开发的我，学习这些底层的东西当然是非常有必要的，应该说非常重要才对

OpenGL Context

  • Context是一个非常抽象的概念，可以尝试将这个理解为一个包含了所有OpenGl状态的对西安；如果把一个Context销毁了，那么OpenGl也将消失
  • OpenGL对象包含了Textures，Vertex Buffer Objects，Shaders，Frame & Render Buffers，State

GLKView

  • 主要功能
    • 配置OpenGL代码来渲染一个视图
    • 设置OpenGL的context
    • 用来重写绘制规则的方法-(void)glkView:(GLKView *)view drawInRect:(CGRect)rect { ... }；在{}当中写入OpenGL代码
  • 相关的属性与方法
    • drawableColorFormat
      • OpenGL上下文有一个缓冲区，它用以存储将在屏幕中显示的颜色。你可以使用其属性来设置缓冲区中每个像素的颜色格式
      • 缺省值是GLKViewDrawableColorFormatRGBA8888，即缓冲区的每个像素使用8个bit（所以每个像素4个字节）。这非常好，因为它给了你提供了最广泛的颜色范围，让你的app看起来更好。
      • 如果你的app允许更小范围的颜色，你可以设置为GLKViewDrawableColorFormatRGB565，从而使你的app消耗更少的资源（内存和处理时间）。
    • drawableDepthFormat
      • 这个缓冲区称之为深度缓冲区，这将会帮助我们确保更加接近观察者的对西安显示在远一些的对象的前面，也就是可以让前面的对象挡住后面的对象

        GLKViewController

  • 每一帧页面的刷新都会调用-(void)update{}方法，可以通过这个方法来实现屏幕每一帧的渲染工作
  • 注意：这个方法在写的时候很可能不会有代码提示，这是Xcode的bug，不用理会，直接写就可以了
  • 这个类抽取出大量的样板代码，这些代码完成了OpenGL ES项目的基本配置

GLKEffects

  • 这些类实现了OpenGL ES1.0公共的shdaing行为，简化了从1.0到2.0的转化；它们也提供了让光和纹理(lignthing and texturing)工作的简单方法

GLMath

  • 在iOS5之前，每个游戏都需要它们自己的数学库，处理公共的向量和矩阵操作方法。现在有了GLMath，大多数的公共方法都可以从GLMath中获得

GLKTextureLoader

  • 这个类使得加载图像作为OpenGL使用的纹理更加简单。和写一个复杂的方法来处理大量不同的图像格式比，现在加载一个纹理只需要一个简单的方法调用

FrameBuffer

  • 这个buffer是其他buffer的集合

  • ColorBuffer

    • 这个可以理解为是一对像素的集合，GLKView会在每一次的渲染当中将这里取出来的像素放到视图当中

      g glClearColor(_red, _green, _blue, _alpha);
glClear(_mask);

    • 第一行代码调用glClearColor定义用来清理屏幕的RGB颜色和alpha值，第二行代码调用glClear来实际执行清理操作，缓冲区类型有很多

使用贝塞尔曲线实现一个弹性的UITextField

前言

  • 这篇文章在看到ray上面的一篇教程之后学到的，看了那篇教程之后，让我对于动画的制作有了一个新的理解，同时这个动画的实现设计并不好，在我自己的尝试过程中发现了一些问题，好了，那么废话不多说了，就让我们开始吧！

实现思路

  • 实现的基本输入其实就是通过贝塞尔曲线画出变化之前与变化之后TextField的形状，然后再使用SpringAnimation来实现形变过程中的弹性动画，最后再使用CADisplayLink来渲染每一帧的形变
  • 将得到的Elastic View作为UITextField的子视图来使用，便可以实现弹性的UITextField

    }

  • 这里首先创建了四个控制点，通过UIView的形式来展示，当然也可以使用CALayer，作为debugPoint
  • 接下来创建了一个CAShapeLayer对象，用于将绘制好的贝塞尔曲线展示出来
  • 最后重写初始化方法，调用setupComponents()，这个方法将会完成动画的基本配置工作

  • 那么现在需要将这四个控制点加入到Elastic View当中，可以通过一个for循环来实现

使用贝塞尔曲线来画出形变前后的TextField

  • 这就是形变之后的TextField

  • 下面的方法将会画出一个形变之后的UITextField

    private func bezierPathForControlPoints()->CGPathRef {
    // 1
    let path = UIBezierPath()

    // 2
    let top = topControlPointView.layer.presentationLayer().position
    let left = leftControlPointView.layer.presentationLayer().position
    let bottom = bottomControlPointView.layer.presentationLayer().position
    let right = rightControlPointView.layer.presentationLayer().position

    let width = frame.size.width
    let height = frame.size.height

    // 3
    path.moveToPoint(CGPointMake(0, 0))
    path.addQuadCurveToPoint(CGPointMake(width, 0), controlPoint: top)
    path.addQuadCurveToPoint(CGPointMake(width, height), controlPoint:right)
    path.addQuadCurveToPoint(CGPointMake(0, height), controlPoint:bottom)
    path.addQuadCurveToPoint(CGPointMake(0, 0), controlPoint: left)

    // 4
    return path.CGPath

}

  • 这个方法追踪四个控制点的位置，然后绘制出每一阶段形变的贝塞尔曲线，形变之前与之后的TextField都是通过这个方法来绘制的，区别就在于控制点位置的变化

  • 那么现在就需要使用CADisplayLink来渲染每一帧的形变了

    private lazy var displayLink : CADisplayLink = {
    let displayLink = CADisplayLink(target: self, selector: Selector("updateLoop"))
    displayLink.addToRunLoop(NSRunLoop.currentRunLoop(), forMode: NSRunLoopCommonModes)
    return displayLink
}()

func updateLoop() {
    elasticShape.path = bezierPathForControlPoints()
}

private func startUpdateLoop() {
    displayLink.paused = false
}

private func stopUpdateLoop() {
    displayLink.paused = true
}

  • 这里新建一个CADisplayLink对象，做了基本的配置，每一帧刷新的时候调用的是updateLoop，在这里面变更elasticShape的形状

  • 然后是displayLink的停止与开启

UIView Spring Animations

  • 那么现在就在给形变过程加入动画效果的时候了

    func animateControlPoints() {
    //1
    let overshootAmount : CGFloat = 10.0
    // 2
    UIView.animateWithDuration(0.25, delay: 0.0, usingSpringWithDamping: 0.9, initialSpringVelocity: 1.5,
    options: nil, animations: {
    // 3
    self.topControlPointView.center.y -= overshootAmount
    self.leftControlPointView.center.x -= overshootAmount
    self.bottomControlPointView.center.y += overshootAmount
    self.rightControlPointView.center.x += overshootAmount

},
completion: { _ in
    // 4
    UIView.animateWithDuration(0.45, delay: 0.0, usingSpringWithDamping: 0.15, initialSpringVelocity: 5.5,
        options: nil, animations: {
            // 5
            self.positionControlPoints()

        },
        completion: { _ in
            // 6
            self.stopUpdateLoop()
        })
    })
}

  • 这样便给了开始形变的过程以及结束形变的过程添加了弹性的动画效果

  • 这里overshootAmount指的是形变度

  • 最后之后重写touchesBegan(touches: Set, withEvent event: UIEvent)方法来开启动画以及启动displaylink

    override func touchesBegan(touches: Set, withEvent event: UIEvent) {
    startUpdateLoop()
    animateControlPoints()
}

优化重构

制作一个Elastic UITextField

  • 新建一个ElasticTextField.swift文件继承自UITextField，然后添加一下内容

    import UIKit

class ElasticTextField: UITextField {

    // 1
    var elasticView : ElasticView!

    // 2
    @IBInspectable var overshootAmount: CGFloat = 10 {
        didSet {
            elasticView.overshootAmount = overshootAmount

        }

    }

    // 3
    override init(frame: CGRect) {
        super.init(frame: frame)
    setupView()

    }

    required init(coder aDecoder: NSCoder) {
        super.init(coder: aDecoder)
        setupView()

    }

    // 4
    func setupView() {
        // A
        clipsToBounds = false
        borderStyle = .None

        // B
        elasticView = ElasticView(frame: bounds)
        elasticView.backgroundColor = backgroundColor
        addSubview(elasticView)

        // C
        backgroundColor = UIColor.clearColor()

        // D
        elasticView.userInteractionEnabled = false

    }

    // 5
    override func touchesBegan(touches: Set, withEvent event: UIEvent) {
        elasticView.touchesBegan(touches, withEvent: event)

    }

}

  • Swift的属性监听确实非常的不错

  • 这里需要注意的是elasticView.userInteractionEnabled = false关闭了elasticView对用户的响应，使用ElasticTextField的响应来调用elasticView的touchesBegan方法(原来touchBegan方法也是可以直接调用的)，这样避免因为响应了elasticView而屏蔽了对ElasticTextField的响应，这样就会造成无法输入的情况；另外还要注意一下clipsToBounds的调整
  • 现在在StroyBoard当中拖拽一个UITextfield并将其类设置为ElasticTextField便可实现如图所示的效果

如何遍历一个非对象数组

  • 不能够使用for-in循环来进行遍历，这个循环在objective-c当中只能够遍历对象数组，比如NSAarray或者是NSMutableArray之类的，但是这些数组都只能够装填对象元素，类似于int、double之类的基本元素是不能装进去的

  • 解决方案是通过普通的for循环来实现遍历

    int array[] = {1,2,3,4,5};
for(int i = 0;i <= 4;i++){
    int temp = array[i];
    //do whatever you like
}

  • 实际上在swift当中for-in循环是可以用来遍历非对象数组的
